# AllyTV Mini Game — 제품 기획서

> **버전:** v0.1 (프로토타입)  
> **작성일:** 2026-03-04  
> **대상 연령:** 유아 (만 3~7세)  
> **플랫폼:** HTML5 웹앱 (Mobile + TV 연동)

---

## 1. 제품 개요

### 1.1 서비스 정의

**AllyTV Mini Game**은 스마트폰(태블릿)과 TV 화면을 동시에 활용하는 **이중 화면 유아 교육 게임**이다.  
아이가 스마트폰 화면을 직접 터치하여 미니게임을 조작하면, TV 화면에는 별도로 설계된 풍부한 연출이 표시되어 부모와 아이가 함께 즐길 수 있다.

### 1.2 핵심 컨셉

| 구분 | 내용 |
|------|------|
| 타겟 유저 | 만 3~7세 유아 (조작 : 터치 중심) |
| 플레이 방식 | 스마트폰 터치 조작 → TV 화면 연출 출력 |
| 교육 목표 | 수 감각(숫자 세기), 비교(길이/수량), 논리적 사고(경로 찾기), 집중력 향상 |
| 게임 수 | 7종 (Ally1 ~ Ally7) |
| 제한 시간 | 게임별 5초 타이머 (유아 집중력 고려) |

### 1.3 기술 스택

- **프론트엔드:** HTML5 + Vanilla CSS + JavaScript (ES6 Class 기반)
- **렌더링:** Canvas 2D API (모바일 캔버스 / TV 캔버스 이중 렌더링)
- **오디오:** Web Audio API (`audio.js`)
- **구조:** SPA (Single Page Application) — 화면 전환은 DOM `active` 클래스 방식

## 2. 시스템 아키텍처

### 2.1 이중 화면 구조

```
┌──────────────────────────┐    ┌─────────────────────────────┐
│       TV 화면 (상단)      │    │   스마트폰 화면 (하단)       │
│  - 별도 TV 전용 연출       │    │  - 실제 조작 인터페이스       │
│  - 게임 결과 시각화        │    │  - 터치 입력 처리            │
│  - 풍부한 애니메이션        │    │  - 타이머 표시              │
└──────────────────────────┘    └─────────────────────────────┘
```

각 게임은 `renderTV()` 메서드를 통해 TV 전용 화면을 독립적으로 렌더링한다.

### 2.2 게임 상태 흐름

```
홈 화면 (게임 선택)
    │
    ▼ 게임 카드 터치
게임 진행 (5초 타이머)
    │
    ├─ 성공 → 성공 연출 → 결과 화면 (홈 / 재시도)
    └─ 실패 또는 타임아웃 → 실패 연출 → 결과 화면
```

### 2.3 공통 게임 인터페이스

모든 게임 클래스는 다음 공통 메서드를 구현한다:

| 메서드 | 역할 |
|--------|------|
| `init()` | 캔버스 컨텍스트 및 이미지 로딩 초기화 |
| `start()` | 게임 시작 (랜덤 레벨 생성, 애니메이션 루프 시작) |
| `onButtonDown(x, y)` | 터치/클릭 입력 처리 |
| `onTimeout()` | 타임아웃 시 실패 처리 |
| `renderTV()` | TV 캔버스 독립 렌더링 |
| `cleanup()` | 게임 종료 및 리소스 정리 |

### 🦷 Game 11 — 치카치카 양치질 빨리하기

**`Ally1_brushing teeth.js`**

#### 개요

양치질을 주제로 한 **연속 터치 게임**. 빠르게 여러 번 터치하여 양치를 완료하는 것이 목표다.

#### 게임 목표 / 교육 요소

- 양치질의 올바른 습관 인식
- 집중력 및 연속 동작 훈련

#### 게임플레이 메커니즘

| 항목 | 내용 |
|------|------|
| 조작 | 화면 아무 곳이나 터치 (탭) |
| 성공 조건 | 7~10회 (랜덤) 터치 달성 |
| 실패 조건 | 5초 타이머 내 목표 터치 수 미달성 |
| 터치 피드백 | 터치마다 모바일 이미지 단계적 변화 (8단계) |

#### 화면 연출

| 화면 | 성공 시 | 실패 시 |
|------|---------|---------|
| 모바일 | 양치 완료 이미지 | 실패 이미지 |
| TV | 단계별 칫솔 진행 + 성공 스파클 효과 | 최종 실패 이미지 |

### 🌸 Game 14 — 꽃병에 꽃 2송이를 찾아보자

**`Ally4_flower.js`**

#### 개요

1송이, 2송이, 3송이가 담긴 꽃병 3개 중, **꽃이 2송이인 꽃병을 터치**하는 수 선택 게임이다.

#### 게임 목표 / 교육 요소

- 수 감각 (1, 2, 3 구분)
- 집합과 수량 인식

#### 게임플레이 메커니즘

| 항목 | 내용 |
|------|------|
| 조작 | 꽃병 이미지 터치 |
| 성공 조건 | 2송이 꽃병 터치 |
| 실패 조건 | 잘못된 꽃병 터치 또는 타임아웃 |
| 난이도 | 꽃병 좌우 배치 및 꽃 이미지 종류 랜덤화 |

#### 화면 연출

| 화면 | 성공 시 | 실패 시 |
|------|---------|---------|
| 모바일 | 선택한 꽃병 + 숫자 이미지 표시 | 선택한 꽃병 + 숫자 이미지 표시 |

### 📮 Game 16 — 편지 부치러 우체국 꼬불꼬불 길 찾기

**`Ally6_maze.js`**

#### 개요

미로 3종 중 하나가 표시되고, **미로 출구와 연결된 캐릭터를 4명 중에서 선택**하는 추리/경로 게임이다.

#### 게임 목표 / 교육 요소

- 논리적 사고 (경로 추적)
- 집중력 및 시각 변별력

#### 게임플레이 메커니즘

| 항목 | 내용 |
|------|------|
| 조작 | 하단 캐릭터 4명 중 1명 터치 |
| 성공 조건 | 해당 미로의 정답 캐릭터 위치 선택 |
| 실패 조건 | 오답 캐릭터 선택 또는 타임아웃 |
| 미로 종류 | 3종 랜덤 출제 (미로별 정답 위치 고정) |
| 캐릭터 배치 | 4명의 캐릭터 순서 랜덤화 |

#### 정답 조건표

| 미로 번호 | 정답 캐릭터 위치 (0-based) |
|-----------|--------------------------|
| 미로 1 | 2번째 (index 1) |
| 미로 2 | 3번째 (index 2) |
| 미로 3 | 1번째 (index 0) |

#### 화면 연출

| 화면 | 성공 시 | 실패 시 |
|------|---------|---------|
| 모바일 | 선택 캐릭터 사라짐 | 선택 캐릭터 사라짐 |
| TV | 캐릭터 중앙 표시 1초 → 우편 배달 인터랙션 슬라이드 | 캐릭터 중앙 표시 1초 → CLOSED 간판 + 리본 등장 |

### 🥕 Game 17 — 농부 할아버지를 도와주자

**`Ally7_carrot.js`**

#### 개요

밭에서 당근을 수확하는 컨셉으로, **10번 터치하여 당근 10개를 모두 뽑는** 연속 인터랙션 게임이다.

#### 게임 목표 / 교육 요소

- 수 세기 (1~10)
- 협동 / 돕기 개념 교육
- 집중력 및 연속 조작 훈련

#### 게임플레이 메커니즘

| 항목 | 내용 |
|------|------|
| 조작 | 화면 아무 곳이나 터치 |
| 성공 조건 | 5초 내 10번 터치 완료 |

| 피드백 | 터치마다 당근 1개씩 뽑히는 이미지 업데이트 (11단계) |

- 게임 시작 전 `waiting` 상태: 농부 캐릭터 + 말풍선 깜박임

- 바구니에 터치마다 당근 이미지 1개씩 쌓임
- 성공 시: 농부 기쁜 캐릭터 + 가득 찬 바구니 표시

| 화면 | 성공 시 | 실패 시 |
|------|---------|---------|
| 모바일 | 최종 당근밭 이미지 | 마지막 진행 상태 동결 |
| TV | 행복한 농부 + 가득 찬 바구니 | 실망한 농부 캐릭터 표시 |

## 4. 공통 UX 정책

### 4.1 타이머

- 모든 게임 공통 **5초 제한**
- 타이머 UI: 원형 프로그레스 바 + 숫자 (모바일 우측 상단, TV 미러링)
- 성공 시 즉시 타이머 정지 (`gameTimer.stop()`)

### 4.2 결과 화면

- 성공/실패 후 자동으로 결과 오버레이 표시
- 결과 화면 버튼: **홈으로 돌아가기** / **다시 시작**

### 4.3 오디오

- 터치 피드백음 (`click`)
- 정답 선택음 (`ding`)
- 성공 BGM / 실패 BGM
- 배경음악: 타이머 틱음 반복 루프

### 4.4 접근성 및 조작

- 터치, 마우스 클릭, 키보드(스페이스바/A키) 모두 지원
- `user-scalable=no` — 유아 오조작 방지
